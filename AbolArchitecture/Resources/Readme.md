# Архитектура

# ViewProperties
    - Ответственость
        Данные которые описывают состояния View компонента (UIView, UITableView, UIViewController)
    
    - Создается
        На основе своих полей 
        
    - Зависиммости
        1) UIKit 
        2) В полях только примитивы UIKitа и Foundation
        3) Пртоколы архитектуры ?
    
# View
    - Ответственость
        Отобразить данные из которых состоит ViewProperties
    
    - Создается
        На основе ViewProperties все что есть во ViewProperties должно быть в View
        
    - Зависиммости
       1) UIKit
       2) ViewProperties
       3) Пртоколы архитектуры


# Updater
    - Ответственость
        На основе кейсов enum State зменяет ViewProperties и измененные ViewProperties передает в View
        через метод update(with: viewProperties)
    
    - Создается
        создается пустым с одним опциональным полем ViewProperties ?
        и с enum State без состояния 
        
    - Зависиммости
        1) UIKit
        2) View
        3) ViewProperties
        4) Пртоколы архитектуры

# Builder
    - Ответственость
        Связать между собой ViewManager и View 
    
    - Создается
        Создается с двумя полями ViewManager и View 
        в момент создания связывает между собой Updater и View метод bind(view:)
        
    - Зависиммости
        1) UIKit ?
        2) View
        3) ViewProperties
        4) Пртоколы архитектуры
        
        
# Feature
    - Ответственость
        Вся логика которая относится к конкретной фиче
        Сервисы обслуживающие логику 
    
    - Создается
        На сонове данных без которых не может быть текущий фичи
        
    - Зависиммости
        1) UIKit
        2) View
        3) ViewProperties
        4) Пртоколы архитектуры
        5) Сервисы которые нужны для реализации логики текущий фичи

# Coordinator
    - Ответственость
        Переход между фичами
        Передача данных между фичами
        Сервисы для реализации логики перехода (Router)
    
    - Создается
        На основе фичей (Фича из которой идет переход) и (Фичи куда идет переход) 
        
    - Зависиммости
        1) Router (нативная логика переходов между VC)
        2) Пртоколы архитектуры
        3) UIKit
        4) Текущая фича

# FlowCoordinator
    - Ответственость
        Обьеденяет Coordinatorы всех фичей проекта
        Устанавливает рутовый VC
    
    - Создается
        На основе Coordinatorов фичей
        
    - Зависиммости
        1) Пртоколы архитектуры
        2) Фичи проекта
